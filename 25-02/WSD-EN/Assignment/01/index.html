Assignment 1: Design an Online Bookstore

Assignment topics

Design an ERD and write an API specification for a book management service

Assignment objectives

  • Understand the service domain structure through database modeling (ERD).
  • Design a request/response structure by writing a RESTful API specification.
  • Gain experience in refining the DB specification and API specification to a level that can be utilized for actual implementation/automation.
  • Complete the design with consistent naming conventions and constraint definitions.
  • You can use either RBD or NoSQL for the DB, but you must write an ERD!

Assignment content

All features should be written with reference to the following site:

1. ERD design

  • Required tables: User, Book, Order, Review, Favorite/Wishlist, Cart
  • Specify PKs for all tables
  • Define FKs and map relationships (design 1:1, 1:N, N:M relationships for your domain)
  • Include constraints (NOT NULL, UNIQUE, CHECK/ENUM, ON DELETE/UPDATE)
  • Review field types for appropriateness (amount=INT/DECIMAL, date=DATETIME, etc.)
  • Include timestamps (created_at, updated_at, deleted_at if necessary)
  • Follow DB naming conventions
    • Use snake cases
    • Do not use meaningless abbreviations or random names

3. Create API specification

  • Design URLs/methods to match RESTful rules
  • For each endpoint, include
    • HTTP methods (GET, POST, PATCH, PUT, DELETE)
    • Status code definitions (200, 201, 400, 401, 403, 404, 409, etc.)
    • Request parameters/Body (JSON standard)
    • Response example (success JSON structure, failure response example if needed)

Features to include

  1. Member management/authentication: signup, login, logout, token renewal, profile view/edit/delete, JWT authentication, bcrypt application
  1. Book management: book registration (admin), list view (pagination/sorting/filtering), detailed view, delete (admin), authors/categories array processing
  1. Review management: create, view, edit (author), add/revoke likes, Top-N cache/performance consideration
  1. Comment management: comment CRUD, like/unlike, add/remove likes
  1. Wishlists/Libraries: add, view, delete
  1. Cart and Orders: (extensible if needed)
  1. Common response formats: Success/Error/Pagination response structure
