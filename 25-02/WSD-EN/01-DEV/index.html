1. Back-End
Back-End is a server-side role that stores, processes, and responds to data and rules in an area that is not directly visible to the user.

A. Definition of Back-End Developer

  • A back-end developer is a role that processes data and implements the core logic of a service in an area that is not directly visible to the user.
  • While the frontend is responsible for the screens of the web and apps, the backend is responsible for the invisible behaviors that validate login results, store posts, and process payments.
  • The backend is therefore the heart and brain of the service.

C. Backend roles

A. User → Frontend

User enters email and password on the login screen and clicks the login buttonThe frontend receives user input data and passes it to the server.

B. Frontend → Backend Server

The front-end deliversthe input data to the back-end server in the form of a login request (Request). This request is mainly sent via HTTP POST, which is secured via HTTPS.

C. Backend Server → Database

When the backend server receives the request, it first checks the database to seeif the entered email actually exists and compares the password hash value.

d. DB result return → Backend server

The database returns the search results to the backend server.

  • If the account does not exist → Prepare to return a failure response
  • If the password does not match → Prepare to return a failure response
  • If it matches → Prepare to return a success response

e. Issuing JWT token

Upon successful authentication, the backend server issues a JWT (Json Web Token) The token contains user identification information, permissions, expiration time, etc. and includes a signature to prevent forgery.

f. Login success response

The backend server delivers the issued token to the frontend, which stores it in browser storage or cookies and includes it in the header of every subsequent request to maintain authentication.

G. User Experience

The user successfully connects to the service with a successful login response and is able to use personalized screens and features.

D. Why You Need a Backend

  • Need to ensure the authenticity and reliability of the information displayed on the screen
  • Persistence requirements for features with statusessuch as login, payment, referrals, notifications, etc.
  • The need to maintain response time and availability with thousands of concurrent users
  • Need for security and privacyto meet legal and ethical requirements

    (Service examples)

    • Instagram scenario: upload photo → save original, generate thumbnail, record post metadata, and update follower feeds
    • Delivery app scenario: Enter address → search for store, check inventory, calculate cart price, authorize payment, call delivery agency
    • Streaming service scenario: user authentication → validate subscription rights, select CDN route, record viewing history, input recommendation model

2. Back-End Developer

A. Main tasks of a backend developer

  • a. Data processing
    • Safely storing information in the database when a membership request comes in
    • Perform authentication by comparing database information with the information entered at the time of login
    • Accurately managing all service data such as posts, comments, order history, payment history, etc.

Example)

  • Storing email and password entered by users in the MySQL users table
  • Encrypt the entered password with bcryptand compare it with the stored hash
  • When a post is created, save it to the posts table and retrieve it with comments by JOINwhen viewing it

  • B. Implement business logic
    • Implementing a procedure to check inventory and record it in the database when a user adds a product to the cart
    • Implement procedures to authorize payment, deduct inventory, create an order, and issue a receipt when clicking the checkout button
    • Writing rules in code for service operation

Example)

  • Check inventory in inventory table and write to cart_items table when "Add to cart" is clicked
  • After payment authorization, create order in orders table, subtract quantity from inventory table, and record receipt in receipts table
  • Implement logic to automatically apply 10% discount for orders placed during certain time periods

  • C. Security and Authentication
    • Ensuring that users have the correct authorization
    • Storing passwords securely and encrypted
    • Blocking access from unauthorized users

Example)

  • Return 403 Forbidden when a non-admin user accesses the /admin endpoint
  • Hash password with bcryptand store it in DB, but not the original password
  • Return 401 Unauthorized response when a non-logged-in user requests the post creation API
  • D. Handling performance and scalability
    • Ensuring stable behavior even when thousands or tens of thousands of users access at the same time
    • Maintaining performance by applying cache, scaling servers, load balancing, etc.

Example)

  • Spreading traffic across multiple servers and using load balancers in a live chat service
  • Reducing DB load by storing a list of popular posts in a Redis cache
  • Balancing server load with Nginx load balancing

B. Major challenges faced by backend developers

  • Performance issues: Server lag, database bottlenecks during high traffic situations
  • Security threats: SQL Injection, XSS, risk of authentication token hijacking
  • Scalability needs: Server scaling and data management issues as the service grows larger
  • Failure response: Have recovery procedures in place in case of server down, data loss, or deployment failure

C. Tools and technology stack used by backend developers

  • Development languages: Java, Python, JavaScript (Node.js), Go
  • Frameworks: Spring Boot, Express, Django
  • Databases: MySQL, PostgreSQL, MongoDB, Redis
  • Operations/deployment: AWS, Docker, Kubernetes, GitHub Actions
  • Collaboration tools: GitHub, Jira, Slack

B. Basic concepts of the web: background to understand the behavior

  • HTTP protocol:

    Communication rules that clients (users) and servers must follow when sending and receiving data.

    Example: When you type the address http://example.com/users in your browser, you send a request to the server; the server responds with an HTML or JSON response
  • Request:

    A message that a user sends to a server, usually consisting of a URL + method (GET, POST, etc.) + header + body

    Example: Email-password entered by the user at login is sent to the server in the body of a POST request
  • Response:

    Message sent back to the user by the server, containing status codes and data (JSON, HTML, XML, etc.)

    Example: Return "Login successful, token issued" message with 200 OK status code on successful login
  • REST API:

    A design approach that divides resources into addresses (URLs) and behaviors into methods (GET, POST, PUT, DELETE).

    Example:
    • GET /users → Get all users
    • POST /users → Register a new user
    • PUT /users/1 → Modify specific user information
    • DELETE /users/1 → Delete a specific user

c. Database: The entity that manages data!

  • Relational Database (RDBMS):

    A table-based database like MySQL, PostgreSQL, etc.

    • Stores data in a row and column structure and manipulates it with SQL
    • Normalizeto reduce duplicates and ensure data integrity
    • JOINto connect and view data from multiple tables
    Example:
    • JOIN theusers table with the orders table to get the order history of a specific user
    • Linking an account table with a transaction history table in a banking system
  • Non-relational databases (NoSQL):

    A free-form way to store data, such as MongoDB and Redis.

    • Document-based storage: MongoDB → flexible data structure in JSON format
    • Key-Value storage: Redis → cache, optimized for session storage
    Example:
    • Storing user profile information from a social network as JSON in MongoDB
    • Store the last 100 live chat room messages in Redis for quick fetching
  • Object Relational Mapping (ORM):

    Tools like JPA, Sequelize, and Django ORMs

    • Manipulate DB with code in object-oriented languages without writing SQL directly
    • Automatically handle DB table ↔ code object mapping
    Example:
    • User.findByEmail("test@test.com") → execute SELECT * FROM users WHERE email = ... automatically without SQL
    • Django ORM to automatically lookup posts table when calling Post.objects.all()

D. Security and authentication: key areas that determine the reliability of your backend

  • Sessions and cookies
    • The server saves the session, and the client keeps a cookie with the session ID.
    • Traditional and intuitive, but can increase server memory/DB load
    Example: Keeping you logged in to a shopping mall
  • JWT (JSON Web Token)
    • Token-based authentication, client carries a token to verify with server
    • Highly scalable as the server only needs to verify the token
    Example: After logging in to a mobile app, pass Authorization: Bearer <token> every time when calling an API
  • OAuth 2.0
    • A way to sign in using an external service account
    • Increases security and simplifies the user experience
    Example: Google/Cakao/Netflix login
  • Password encryption
    • Hash algorithms like BCrypt and Argon2 prevent the original password from being stored
    Example: If a user forgets their password, do not restore it, provide a reset link

e. Deployment and operations: the final responsibility of the backend to keep the service alive

  • Linux Basic Commands
    • File manipulation, permission setting, network management commands essential in a server environment
    Examples: cd, ls, chmod, systemctl
  • Cloud Services (AWS, GCP, Azure)
    • Provide server infrastructure, databases, storage, and networking
    Examples: Deploying servers on AWS EC2, operating databases with RDS
  • Docker
    • Packaging execution environments into containers → Ensure the same environment everywhere
    Example: Solving the "it works on my PC, but not on the server" problem by keeping local development and deployment environments the same
  • CI/CD
    • Automated build-test-deploy pipeline for code changes
    Example: GitHub Actions → Automatically push to server when merged into main branch
  • Monitoring tools
    • Real-time observation of server health, traffic, and error logs
    Example: Monitoring server CPU/memory with Grafana, searching error logs with ELK stack
