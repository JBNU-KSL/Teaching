1.1 최적화 문제의 정식화

A. 일반적 최적화 문제:

  • 목적함수 (Objective function)
    • f(x)f(x)f(x)﻿: 우리가 최소화(또는 최대화)하고자 하는 함수.
    • 실제 문제에서는 비용(cost), 에너지, 거리, 오차(error) 등 다양한 형태로 정의됨.
  • 변수 공간 (Decision/phenotype space)
    • x∈Xx \in \mathcal{X}x∈X﻿: 최적화를 수행하는 대상 변수.
    • X\mathcal{X}X﻿는 문제의 해 공간(feasible search space)으로, 연속 공간(예: Rn\mathbb{R}^nRn﻿), 이산 공간(예: 조합적 해), 또는 순열 구조(TSP 등)일 수 있음.
  • 부등식 제약 (Inequality constraints)
    • gi(x)≤0,  i=1,…,mg_i(x) \leq 0, \; i=1,\dots,mgi​(x)≤0,i=1,…,m﻿
    • 물리적, 자원적, 설계상의 제한조건을 나타냄.
    • 예: “사용되는 자원 ≤ 허용량”, “압력 ≤ 한계치”.
  • 등식 제약 (Equality constraints)
    • hj(x)=0,  j=1,…,ph_j(x) = 0, \; j=1,\dots,phj​(x)=0,j=1,…,p﻿
    • 반드시 만족해야 하는 강한 제약조건.
    • 예: “수요 = 공급”, “전하 보존”.
  • 해 공간 (Feasible set)
    • 모든 제약조건을 만족하는 해들의 집합 F={x∈X:gi(x)≤0,hj(x)=0}\mathcal{F} = \{ x \in \mathcal{X} : g_i(x) \le 0, h_j(x)=0 \}F={x∈X:gi​(x)≤0,hj​(x)=0}﻿
    • 최적화 알고리즘은 이 유효해 집합(feasible region) 위에서 f(x)f(x)f(x)﻿를 최소화하는 x∗x^*x∗﻿를 찾음.

B. GA에서의 최적화 문제

  • 현상공간(phenotype): X\mathcal{X}X﻿ (연속/이산/순열 등).
  • 유전자공간(genotype): g \mathcal{g}g﻿ (비트열, 실수 벡터, 순열 등).
  • 표현/부호화(encoding) 사상: φ:g→X\varphi: \mathcal{g} \to \mathcal{X}φ:g→X﻿.
  • f(φ(g))f(\varphi(g))f(φ(g))﻿ : 현상공간에서 정의된 목적함수
  • GA는 G\mathcal{G}G﻿에서 표본추출과 재조합을 반복하고, 평가는 F(g)=f(φ(g))F(g)=f(\varphi(g))F(g)=f(φ(g))﻿ 로 수행.

C. 핵심 개념

  • 국소성 (Locality)
    • 유전자공간의 거리 dG(g1,g2)d_\mathcal{G}(g_1,g_2)dG​(g1​,g2​)﻿가 현상공간의 거리 dX(φ(g1),φ(g2))d_\mathcal{X}(\varphi(g_1), \varphi(g_2))dX​(φ(g1​),φ(g2​))﻿와 잘 상관될수록 탐색이 효율적임
  • 중복성 (Redundancy)
    • φ\varphiφ﻿가 비단사일 때, 서로 다른 유전자가 같은 해로 매핑되는 중복이 발생
    • 불필요한 탐색 낭비를 유발할 수 있음
  • 상호작용 (Epistasis)
    • 유전자좌(loci) 간의 상호작용 정도
    • Walsh 변환, NK-랜드스케이프 모델 등을 통해 정량화
      • Walsh 변환: 이진 유전자열에 정의된 함수를 직교 기저(±1 다항식)로 분해하여, 어떤 차수의 상호작용(에피스타시스)이 강한지 스펙트럼 형태로 보여주는 도구.
      • NK-랜드스케이프 모델: $N$개의 유전자좌가 있고 각 좌가 KKK﻿개의 다른 좌와 상호작용한다고 가정하여, 문제의 난이도(러기드니스)를 조절할 수 있는 랜덤 적합도 풍경 생성 모델.
    • 강한 에피스타시스는 GA 탐색을 어렵게 만듦
  • 조작 닫힘성 (Operator closure)
    • 선택·교차·돌연변이 같은 연산 후에도 항상 유효한 해를 생성하는가?
    • 예: 순열 문제에서 교차 후에도 유효한 순열을 유지해야 함

1.2 탐색공간과 적합도 풍경(Fitness Landscape)

  • 그래프/메트릭 공간 시점
    • 탐색 공간을 (G,N)(\mathcal{G}, \mathcal{N})(G,N)﻿으로 설정

    • $\mathcal{N}$: 이웃(neighborhood) 연산자 집합 (예: 비트플립, swap, 2-opt 등)

  • 러기드니스 (Ruggedness)
    • 적합도 풍경이 얼마나 “울퉁불퉁”한지를 나타내는 지표.
    • 무작위 보행(random walk): g0,g1,…,gTg_0, g_1, \dots, g_Tg0​,g1​,…,gT​﻿ 위에서 적합도 시퀀스 Ft=F(gt)F_t = F(g_t)Ft​=F(gt​)﻿를 얻음.
    • 자기상관 함수 (autocorrelation function):
    • 자기상관 길이 (autocorrelation length, ℓac\ell_{ac}ℓac​﻿):
      • 무작위 보행(random walk)에서 한 해와 그 다음 해의 적합도 간 상관관계가 얼마나 빨리 사라지는지 측정.
      • ℓac\ell_{ac}ℓac​﻿가 길면 풍경이 매끄럽고 탐색이 쉬움 (smooth landscape), 짧으면 러기드하고 지역최적이 많음 (rugged landscape) → 지역최적(local optima)이 많고 탐색 어려움.

  • 디셉션 (Deception)
    • 지역적 탐색 정보가 전역 최적해로 이어지지 않고, 오히려 반대로 유도되도록 의도적으로 설계된 함수.

    • 대표적 예: trap-kkk﻿ 함수
      • uuu﻿ = 비트열 내 1의 개수
      • u<ku < ku<k﻿일 때는 0k0^k0k﻿ 방향이 점점 더 좋은 해로 보이도록 설계
      • 하지만 전역 최적은 1k1^k1k﻿ → GA가 잘못된 방향으로 수렴할 위험 존재
    • GA 알고리즘의 강건성(stress test)을 평가할 때 사용됨.
