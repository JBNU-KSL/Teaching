01. Back-End Developer

날짜		25.09.04
주차		2주차
1. Back-End
백엔드란? 사용자가 직접 보지 못하는 영역에서 데이터를 저장하고, 가공하고, 규칙에 따라 응답하는 서버 측 역할

A. 백엔드 개발자의 정의

  • 백엔드 개발자는 사용자가 직접 보지 못하는 영역에서 데이터를 처리하고 서비스의 핵심 로직을 구현하는 역할 담당자
  • 웹과 앱의 화면은 프론트엔드가 담당하지만, 로그인 결과를 검증하고 게시글을 저장하며 결제를 처리하는 보이지 않는 동작은 백엔드 담당자
  • 따라서 백엔드는 서비스의 심장과 두뇌에 해당하는 영역

B. 프론트엔드 vs 백엔드

구분프론트엔드백엔드
보이는 것버튼, 화면, 이미지직접 보이지 않는 영역
주요 도구HTML, CSS, ReactJava, Python, Node.js, Spring
역할UI/UX, 사용자 입력 처리데이터 처리, 인증, 보안, 비즈니스 로직
예시로그인 버튼 클릭로그인 검증, DB 확인, 토큰 발급

C. 백엔드의 역할

a. 사용자 → 프론트엔드

사용자가 로그인 화면에서 이메일과 비밀번호를 입력하고 로그인 버튼을 클릭하는 단계
프론트엔드는 사용자의 입력 데이터를 받아서 서버로 전달하는 역할 수행

b. 프론트엔드 → 백엔드 서버

프론트엔드는 입력값을 로그인 요청(Request) 형태로 백엔드 서버에 전달하는 단계
이 요청은 주로 HTTP POST 방식으로 전송되며, HTTPS를 통해 보안이 보장되는 방식

c. 백엔드 서버 → 데이터베이스

백엔드 서버는 요청을 받으면 먼저 데이터베이스를 조회하는 단계
입력받은 이메일이 실제로 존재하는지 확인하고, 비밀번호 해시값을 비교하는 과정 수행

d. DB 결과 반환 → 백엔드 서버

데이터베이스는 조회 결과를 백엔드 서버로 반환하는 단계

  • 존재하지 않는 계정일 경우 → 실패 응답 반환 준비
  • 비밀번호가 일치하지 않을 경우 → 실패 응답 반환 준비
  • 일치할 경우 → 성공 응답 준비

e. JWT 토큰 발급

인증에 성공하면 백엔드 서버는 JWT(Json Web Token) 을 발급하는 단계
토큰 안에는 사용자 식별 정보, 권한, 만료 시간 등이 담겨 있으며 서명(Signature)으로 위조 방지 기능이 포함

f. 로그인 성공 응답

백엔드 서버는 발급한 토큰을 프론트엔드에 전달하는 단계
프론트엔드는 이 토큰을 브라우저 저장소나 쿠키에 저장하고, 이후 요청마다 헤더에 포함시켜 인증을 유지

g. 사용자 경험

사용자는 로그인 성공 응답을 통해 서비스에 정상적으로 접속하게 되고, 이후 개인화된 화면이나 기능을 사용할 수 있는 단계

D. 백엔드가 필요한 이유

  • 화면에 표시되는 정보의 진짜 근원과 신뢰성 보장 필요성
  • 로그인, 결제, 추천, 알림 등 상태를 가진 기능의 지속성 요구사항
  • 수천 명 동시 접속 상황에서 응답 시간과 가용성 유지 필요성
  • 법적·윤리적 요구사항을 만족시키는 보안과 개인정보 보호의 필수성

    (서비스 예시)

    • 인스타그램 시나리오: 사진 업로드 → 원본 저장, 썸네일 생성, 포스트 메타데이터 기록, 팔로워 피드 갱신 절차
    • 배달앱 시나리오: 주소 입력 → 상점 검색, 재고 확인, 장바구니 가격 계산, 결제 승인, 배달 대행사 호출 절차
    • 스트리밍 서비스 시나리오: 사용자 인증 → 구독 권한 검증, CDN 경로 선택, 시청 이력 기록, 추천 모델 입력 절차

2. Back-End Developer

A. 백엔드 개발자가 담당하는 주요 업무

  • a. 데이터처리
    • 회원가입 요청이 들어왔을 때 데이터베이스에 정보를 안전하게 저장하는 업무
    • 로그인 시 입력한 정보와 데이터베이스 정보를 비교하여 인증을 수행하는 업무
    • 게시글, 댓글, 주문 내역, 결제 기록 등 모든 서비스 데이터를 정확하게 관리하는 업무

예시)

  • 사용자가 입력한 이메일·비밀번호를 MySQL users 테이블에 저장
  • 입력한 비밀번호를 bcrypt로 암호화 후 저장된 해시와 비교
  • 게시글 작성 시 posts 테이블에 저장 후 조회 시 JOIN으로 댓글 함께 불러오기

  • b. 비지니스 로직 구현
    • 사용자가 장바구니에 상품을 담으면 재고를 확인하고 데이터베이스에 기록하는 절차 구현 업무
    • 결제 버튼 클릭 시 결제 승인, 재고 차감, 주문 생성, 영수증 발행 절차 구현 업무
    • 서비스 운영을 위한 규칙을 코드로 작성하는 업무

예시)

  • “장바구니 담기” 클릭 시 inventory 테이블에서 재고 확인 후 cart_items 테이블에 기록
  • 결제 승인 후 orders 테이블에 주문 생성, inventory 테이블에서 수량 차감, receipts 테이블에 영수증 기록
  • 특정 시간대 주문 시 자동으로 10% 할인 적용 로직 구현

  • c. 보안 및 인증
    • 사용자가 올바른 권한을 가지고 있는지 확인하는 업무
    • 비밀번호를 안전하게 암호화하여 저장하는 업무
    • 권한 없는 사용자의 접근을 차단하는 업무

예시)

  • 관리자가 아닌 사용자가 /admin 엔드포인트에 접근 시 403 Forbidden 반환
  • 비밀번호를 bcrypt로 해시 후 DB에 저장, 원본 비밀번호는 저장하지 않음
  • 로그인하지 않은 사용자가 글 작성 API 요청 시 401 Unauthorized 응답 반환
  • d. 성능과 확장성 처리
    • 수천 명, 수만 명의 사용자가 동시에 접근해도 안정적인 동작을 보장하는 업무
    • 캐시 적용, 서버 확장, 로드밸런싱 등을 통해 성능을 유지하는 업무

예시)

  • 실시간 채팅 서비스에서 서버 여러 대를 두고 로드밸런서를 통해 트래픽 분산
  • 인기 게시글 목록을 Redis 캐시에 저장해 DB 부하 줄이기
  • Nginx 로드밸런싱으로 서버 부하 균형 유지

B. 백엔드 개발자가 직면하는 주요 도전 과제

  • 성능 문제: 대규모 트래픽 상황에서 서버 지연, 데이터베이스 병목
  • 보안 위협: SQL Injection, XSS, 인증 토큰 탈취 위험
  • 확장성 요구: 서비스가 커질수록 발생하는 서버 확장과 데이터 관리 문제
  • 장애 대응: 서버 다운, 데이터 유실, 배포 실패 시 복구 절차 마련

C. 백엔드 개발자가 사용하는 도구와 기술 스택

  • 개발 언어: Java, Python, JavaScript(Node.js), Go
  • 프레임워크: Spring Boot, Express, Django
  • 데이터베이스: MySQL, PostgreSQL, MongoDB, Redis
  • 운영/배포: AWS, Docker, Kubernetes, GitHub Actions
  • 협업 도구: GitHub, Jira, Slack

3. How to become a Back-End Developer?

A. 백엔드 개발자가 학습해야하는 기초 지식

a. 프로그래밍 언어: 가장 기초가 되는 지식!

  • Java: 안정성과 대규모 시스템에 강점인 언어
  • Python: 문법이 단순하고 생산성이 높은 언어
  • JavaScript(Node.js): 프론트엔드와 백엔드를 하나의 언어로 개발할 수 있는 언어
  • Go, Kotlin: 최근 트렌드에서 사용 비중이 증가하는 언어

프로그래밍 언어는 단순한 도구이므로 특정 언어 선택보다 기본 원리를 이해하는 태도가 중요

언어특징실제 활용 예시강점
Java안정성과 성능이 뛰어나고 대규모 시스템에 강함은행 결제 시스템, 기업 ERP, 안드로이드 앱 서버대규모 생태계, Spring Boot 등 풍부한 프레임워크
Python문법이 단순하고 가독성이 높아 초보자 친화적스타트업 프로토타입, 머신러닝 추천 시스템, 자동화 도구생산성 높음, 데이터·AI 분야와 연계 강점
JavaScript (Node.js)프론트엔드와 백엔드를 하나의 언어로 개발 가능실시간 채팅 서버, 웹 서비스 초기 버전비동기 처리 강점, 방대한 NPM 생태계
Go문법 단순 + 높은 성능, 동시성 처리에 최적화클라우드 인프라, 마이크로서비스, Docker 개발빠른 컴파일 속도, 경량 서버 제작 적합
KotlinJava와 100% 호환, 간결하고 안전한 문법안드로이드 서버, Spring Boot 기반 서비스코드 간결성, 최신 문법 지원, 모바일 친

b. 웹의 기본 개념: 동작을 이해하기 위한 배경

  • HTTP 프로토콜:

    클라이언트(사용자)와 서버가 데이터를 주고받을 때 지켜야 하는 통신 규칙

    예시: 브라우저에서 http://example.com/users 주소를 입력하면 서버에 요청 전송, 서버는 HTML 또는 JSON 응답
  • 요청(Request):

    사용자가 서버에 보내는 메시지, 주로 URL + 메서드(GET, POST 등) + 헤더 + 바디로 구성

    예시: 로그인 시 사용자가 입력한 이메일·비밀번호가 POST 요청 바디에 담겨 서버로 전송
  • 응답(Response):

    서버가 사용자에게 돌려주는 메시지, 상태 코드와 데이터(JSON, HTML, XML 등) 포함

    예시: 로그인 성공 시 200 OK 상태 코드와 함께 “로그인 성공, 토큰 발급” 메시지 반환
  • REST API:

    자원을 주소(URL)로 구분하고, 메서드(GET, POST, PUT, DELETE)로 동작을 나누는 설계 방식

    예시:
    • GET /users → 모든 사용자 조회
    • POST /users → 새 사용자 등록
    • PUT /users/1 → 특정 사용자 정보 수정
    • DELETE /users/1 → 특정 사용자 삭제

c. 데이터베이스: 데이터를 관리하는 주체!

  • 관계형 데이터베이스 (RDBMS):

    MySQL, PostgreSQL 같은 테이블 기반 데이터베이스

    • 행(Row)과 열(Column) 구조로 데이터를 저장하고 SQL로 조작
    • 정규화를 통해 중복을 줄이고 데이터 무결성 보장
    • JOIN으로 여러 테이블의 데이터를 연결해 조회 가능
    예시:
    • users 테이블과 orders 테이블을 JOIN해서 특정 사용자의 주문 내역 가져오기
    • 은행 시스템에서 계좌 테이블과 거래 내역 테이블 연결
  • 비관계형 데이터베이스 (NoSQL):

    MongoDB, Redis 같은 자유로운 데이터 저장 방식

    • 문서(Document) 기반 저장: MongoDB → JSON 형태로 유연한 데이터 구조
    • Key-Value 저장: Redis → 캐시, 세션 저장에 최적화
    예시:
    • SNS에서 사용자 프로필 정보를 MongoDB에 JSON으로 저장
    • 실시간 채팅방 메시지 최근 100개를 Redis에 보관해 빠르게 불러오기
  • ORM (Object Relational Mapping):

    JPA, Sequelize, Django ORM 같은 도구

    • 객체지향 언어의 코드로 SQL을 직접 쓰지 않고 DB 조작
    • DB 테이블 ↔ 코드 객체 매핑 자동 처리
    예시:
    • User.findByEmail("test@test.com") → SQL 없이 자동으로 SELECT * FROM users WHERE email = ... 실행
    • Django ORM으로 Post.objects.all() 호출 시 자동으로 posts 테이블 조회

d. 보안과 인증: 백엔드의 신뢰성을 좌우하는 핵심 영역

  • 세션과 쿠키
    • 서버에서 세션을 저장하고, 클라이언트는 세션 ID가 담긴 쿠키를 보관
    • 전통적이고 직관적인 방식이지만, 서버 메모리/DB 부하 증가 가능
    예시: 쇼핑몰 로그인 상태 유지
  • JWT (JSON Web Token)
    • 토큰 기반 인증, 클라이언트가 토큰을 들고 다니며 서버 검증
    • 서버는 토큰만 확인하면 되므로 확장성이 높음
    예시: 모바일 앱 로그인 후, API 호출 시 매번 Authorization: Bearer <token> 전달
  • OAuth 2.0
    • 외부 서비스 계정을 이용해 로그인하는 방식
    • 보안성을 높이고, 사용자 경험을 단순화
    예시: 구글/카카오/네이버 로그인
  • 비밀번호 암호화
    • BCrypt, Argon2 같은 해시 알고리즘을 통해 원본 비밀번호 저장 금지
    예시: 사용자가 비밀번호를 잊어도 복원하지 않고, 재설정 링크 제공

e. 배포와 운영: 서비스가 살아있도록 유지하는 백엔드의 마지막 책임

  • 리눅스 기초 명령어
    • 서버 환경에서 필수적인 파일 조작, 권한 설정, 네트워크 관리 명령어
    예시: cd, ls, chmod, systemctl
  • 클라우드 서비스 (AWS, GCP, Azure)
    • 서버 인프라, 데이터베이스, 스토리지, 네트워크 제공
    예시: AWS EC2에 서버 배포, RDS로 데이터베이스 운영
  • Docker
    • 실행 환경을 컨테이너 단위로 패키징 → 어디서든 동일한 환경 보장
    예시: 로컬 개발 환경과 배포 환경을 동일하게 유지해 "내 PC에선 되는데 서버에서 안돼요" 문제 해결
  • CI/CD
    • 코드 변경 시 자동 빌드·테스트·배포까지 이어지는 자동화 파이프라인
    예시: GitHub Actions → main 브랜치에 merge되면 자동으로 서버에 반영
  • 모니터링 도구
    • 서버 상태, 트래픽, 에러 로그를 실시간 관찰
    예시: Grafana로 서버 CPU/메모리 모니터링, ELK 스택으로 에러 로그 검색
