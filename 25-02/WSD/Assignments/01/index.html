과제 1: 온라인 서점 설계

과제 주제

도서 관리 서비스의 ERD 설계 및 API 명세서 작성

과제 목적

  • 데이터베이스 모델링(ERD)을 통해 서비스 도메인 구조를 이해한다.
  • RESTful API 명세서를 작성하여 요청/응답 구조를 설계한다.
  • DB 명세서와 API 명세서를 실제 구현/자동화에 활용 가능한 수준으로 구체화하는 경험을 쌓는다.
  • 일관된 네이밍 규칙과 제약조건 정의를 통해 설계의 완성도를 높인다.
  • DB는 RBD또는 NoSQL 중 어떤 것을 사용해도 상관 없지만, ERD는 반드시 작성해야 합니다!

과제 내용

모든 기능은 다음 사이트를 참고해서 작성하시면 됩니다.

1. ERD 설계

  • 필수 테이블: User, Book, Order, Review, Favorite/Wishlist, Cart
  • 모든 테이블에 PK 명시
  • FK 정의 및 관계 매핑 (1:1, 1:N, N:M 관계를 도메인에 맞게 설계)
  • 제약조건 포함 (NOT NULL, UNIQUE, CHECK/ENUM, ON DELETE/UPDATE)
  • 필드 타입 적절성 검토 (금액=INT/DECIMAL, 날짜=DATETIME 등)
  • 타임스탬프 포함 (created_at, updated_at, 필요시 deleted_at)
  • DB 네이밍 규칙 준수
    • 스네이크 케이스 사용
    • 의미 없는 약어나 임의명 사용 금지

※ 참고 사이트

  • dbdiagram.io
  • erdcloud.com

2. DB 명세서 작성

  • 각 테이블별 컬럼명, 타입, 제약조건, 설명을 표로 정리
  • ERD와 일치하는 필드 구성

3. API 명세서 작성

  • RESTful 규칙에 맞는 URL/메서드 설계
  • 각 엔드포인트에 대해 다음 포함:
    • HTTP 메서드 (GET, POST, PATCH, PUT, DELETE)
    • 상태코드 정의 (200, 201, 400, 401, 403, 404, 409 등)
    • 요청 파라미터/Body (JSON 표준)
    • 응답 예시 (성공 JSON 구조, 필요시 실패 응답 예시)
  • 예시 파일 참고하세요~!

반드시 포함할 기능

  1. 회원 관리/인증: 회원가입, 로그인, 로그아웃, 토큰 갱신, 프로필 조회/수정/삭제, JWT 인증, bcrypt 적용
  1. 도서 관리: 도서 등록(관리자), 목록 조회(페이지네이션/정렬/필터링), 상세조회, 수정/삭제(관리자), authors/categories 배열 처리
  1. 리뷰 관리: 작성, 조회, 수정/삭제(작성자 본인), 좋아요 등록/취소, Top-N 캐시/성능 고려
  1. 댓글 관리: 댓글 CRUD, 좋아요 등록/취소
  1. 위시리스트/라이브러리: 추가, 조회, 삭제
  1. 장바구니 및 주문: (필요 시 확장 가능)
  1. 공통 응답 포맷: 성공/에러/페이지네이션 응답 구조
