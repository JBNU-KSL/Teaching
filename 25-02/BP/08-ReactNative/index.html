08. React Native

날짜

1. Introduction

A. Installing Requirement

  • React native 앱은 js로만 이루어지지 않음
  • 앱에서 코드는 상대적으로 중요하지 않은 부분임
  • React native 앱에서 가장 중요한 부분은 Bridge를 통해서 코드가 운영체제와 통신을 할 수 있도록 하는 것
  • 즉 React native 앱을 다운 받을 때 js코드만 받는 것이 아닌 모든 기본시설이 있는 앱을 다운 받는 것
  • >> 그렇기에 Xcode와 java가 필요한 것

  • JS와 markup/styling을 제외하고 핸드폰에 설치할 앱은 기본시설을 모두 가짐
  • >그래서 컴퓨터와 핸드폰의 앱을 연결시켜서 코드를 전송시킴 (즉 컴파일할 필요 없음!)

B. React Native의 동작 과정

  • ReactJS 웹사이트를 만들 때는 HTML을 만들고 ReactJS로 이동해서 브라우저가 실행됨
  • React Native는 브라우저가 없음, Native는 인터페이스로 코드와 운영체제 사이 번역기 같은 존재임
  • React Native는 운영체제(ios&안드로이드)에게 버튼을 표시해달라는 요청을 함
  • 버튼을 만드는 것은 운영체제의 역할이기에 ios와 안드로이드가 달리 표시됨
  • React Native는 버튼을 만들지 않음, 그저 버튼을 만들어달라고 운영체제에게 메세지를 보낼 뿐

  • native와 bridge 부분에서 우리는 그 어떤 코드도 작성할 필요 없음! 우리는 JS부분만 코드 작성!
  • 동작과정:
    • 이벤트(ex. 버튼을 누른다)가 발생하면 native(운영체제)에서 기록함(화면 어디에서 event가 발생했나?, 눌려진 시간이 어느 정도인가?)
    • -> 그러면 native는 그 정보를 가지고 JSON 메세지를 생성함
    • -> 우리의 코드는 그 메세지를 받게됨
    • -> JS는 이벤트가 발생할 때 실행할 동작(코드)을 운영체제에 보냄
    • -> native에서 메세지를 받아서 UI를 업데이트함

2. Rule of Native

A. native는 웹사이트가 아님, 즉 HTML이 아니기에 div는 사용할 수 없음!

  • 대신 View를 사용하여, 항상 import를 해줘야함!
import { StyleSheet, Text, View } from 'react-native';
  • React Native에 있는 모든 text는 text component 에 들어가야함! span, p 같은 것 사용불가
  • View 내 style이 존재, 그런데 웹에서 사용하던 style 일부 사용 불가(CSS의 98% 사용 가능)
const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#fff',
    alignItems: 'center',
    justifyContent: 'center',
  },
});
  • StyleSheet.create는 객체를 생성하는데 사용!
  • StyleSheet.create를 사용하는 이유는 좋은 자동완성 기능을 제공하기 때문

B. React Native Packages

  • https://reactnative.dev/ 
import { StatusBar } from 'expo-status-bar';
import { StyleSheet, Text, View } from 'react-native';
  • 다른 컴포넌트와 다르게 따로 import 해서 사용하며, 과거에는 현재보다 많은 컴포넌트 존재.
  • 하지만 몇몇 컴포넌트는 정돈되지 않아서 수 많은 버그 발생 -> 컴포넌트를 중요한 기능만 남겨둠

C. Component vs API

  • 컴포넌트는 화면에 렌더링할 항목 (return 안에 있는 코드들을 렌더링함)
  • API는 자바스크립트 코드, 즉 운영체제와 소통하는 역할
  • > 요점은 컴포넌트와 API를 이용해서 폰과 앱이 작동하는 방식을 변경할 수 있다는 것
  • 사라진 컴포넌트를 사용하기 위해서 React Native에서 찾아보면 커뮤니티 패키지를 사용하라고 알려줌
  • React Native Directory 커뮤니티 패키지에는 API와 third-party package 가 존재
  • (third-party package는 기본 API이외에 기업들이 만든 API, 많은 옵션을 제공, 하지만 만약에 문제가 존재한다면 커뮤니티가 고쳐지길 기다려야함(만든 사람이 바쁠수도 있다)(커뮤니티에 의존하는 문제))
  • Expo는 React Native가 지원하지 않는 몇몇 package가 매우 중요하다는 것을 깨닫고, 자체적으로 packages와 API를 만들기 시작함 -> Expo SDK 에서 찾을 수 있음
  • > 그래서 'expo-status-bar'가 존재했던 것
  • React Native가 제공하는 API를 일부 수정해서 커뮤니티에 제공한 것(함수 이름이 다르다거나 차이점 존재)
  • Expo에는 수많은 유용한 패키지가 존재함(지문인식, 구글로그인 등등등)

D. Layouts

export default function App(){

    </View>

FlexBox?

  • 유동적으로 각 요소들을 배치하는 기법(다양한 전자기기는 각 화면 크기가 다른데, 각 화면에 맞는 배치를 일일히 계산할 필요 없이 Flexbox로 구현 가능)
  • 웹에서는 flexbox를 적용하고 싶은 요소의 부모요소(flex container)에 다음 속성 추가
    display : flex;
    import React from "react";
import { View } from "react-native";

export default function App(){
  return (
    <View>
      <View style={{width:100,height:100, backgroundColor:"tomato"}}></View>
      <View style={{width:100,height:100, backgroundColor:"teal"}}></View>
      <View style={{width:100,height:100, backgroundColor:"orange"}}></View>
    </View>
  );
}
  • 웹과의 차이점
    • View에 flexContainer가 되라고 말할 필요가 없음
    • 기본적으로 모든 View는 flexContainer임
    • 웹과 다르게 모바일에서는 기본값이 Column임(웹에서는 Row). 물론 원하는 경우 가로로 사용 가능 <View style={{flexDirection:"row"}}>
    • 만약 오버플로우가 발생하여 요소들이 화면 바깥으로 나가도 현재 상태에서는 스크롤할 수 없음!

E. 반응형 디자인

  • 앱을 보는 화면은 매우 크기가 다양하기에 너비와 높이에 기반하여 레이아웃을 만들지 않음 (width, height, px단위 사용 X)
  • React Native에서는 비율로 레이아웃을 분배함
    import React from "react";
import { View } from "react-native";

export default function App(){
  return (
    <View style={{flex:1}}>
      <View style={{flex:1,backgroundColor:"tomato"}}></View>
      <View style={{flex:2,backgroundColor:"teal"}}></View>
      <View style={{flex:1,backgroundColor:"orange"}}></View>
    </View>
  );
}
